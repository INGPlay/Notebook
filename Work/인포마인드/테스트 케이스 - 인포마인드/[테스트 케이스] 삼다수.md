## \[테스트 케이스\] 삼다수

<br>

```
코드 수정 후 확인해야 할 기능 목록
```

<br>

- 주문이 잘되는지 확인
    - 주문 후에 주문조회 시 동기화가 잘되는지 확인

<br>

- 주문 조회가 잘되는지 확인
    - 주문 조회 페이지에 들어갈 경우 동기화가 되는지 확인
    - ~~페이징 버튼을 클릭할 시 동기화가 진행되는지 확인~~
    - 조회 버튼 클릭 시 동기화가 진행되는지 확인
    - 동기화 시 오류가 나지 않는지 확인

<br>

- 주문 수정이 잘되는지 확인
    - 수정 후에 주문조회 시 동기화가 잘되는지 확인

<br>

- 주문 취소가 잘되는지 확인
    - 삭제 후 주문조회 시 동기화가 잘되는지 확인
    - 취소된 주문이 주문취소내역 조회 순서에 맞게 표시되는지 확인

<br>

- ERP에서 취소가 됐을 때 동기화되는지 확인
    - 사이트 UI가 아닌 API 요청을 통해서 삭제 
        - API 요청을 통해서 삭제 시 삭제사유가 ‘관리자에 의해 
    - 주문조회 시 동기화 되는지 확인
    - 취소된 주문이 주문취소내역의 맨 위에 표시되는지 확인

<br>

- 거래명세서 조회가 잘되는 지 확인
    - 총 명세서건수, 총 판매금액, 총 부가세, 총 판매금액계의 값이 조회된 명세서의 합과 같은지 확인
    - 날짜 수정 후 조회 버튼을 눌렀을 시 페이징이 1인지 확인
    - 날짜 수정 후 조회 버튼을 눌렀을 시 동기화가 진행되는지 확인

<br>

- 상시 동기화 함수 실행 시 에러가 나는 상황이 있는지 확인
    - 테스트 컨트롤러 활용

<br>

- 비밀번호 수정이 잘되는지 확인
    - ERP 로그인 에러 시 CMS에서 로그인 설정(강제로 ERP와 비밀번호를 동기화) 후 다시 테스트
    - 비밀번호 변경 시 DB에서 LOCK\_CNT = NULL, PASSWORD\_UPD\_DT = {로그인 시간}, LAST\_LOGIN\_DT = {로그인 시간) 으로 각각 갱신되는지 확인
    - 수정된 비밀번호로 로그인이 되는지 확인
    - 수정된 비밀번호에서 다시 비밀번호 수정이 되는지 확인
