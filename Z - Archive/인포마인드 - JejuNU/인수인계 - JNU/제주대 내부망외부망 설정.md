## 제주대 내부망/외부망 설정

참고 : [https://blog.naver.com/devch/220805952113](https://blog.naver.com/devch/220805952113)

<br>

- 무선랜
    - 게이트웨이 : 192.168.0.1

<br>

- 유선랜
    - 게이트웨이 : 172.18.4.1

<br>

```
:: 기존 라우트 테이블 삭제
route -f

:: 외부망 연결
route add 0.0.0.0 mask 0.0.0.0 192.168.0.1 -p

::내부망 연결
route add 172.18.4.0 mask 255.255.255.0 172.18.4.1 -p

:: SVN 연결
route add 172.18.0.0 mask 255.255.0.0 172.18.4.1 -p

:: DB 연결
route add 203.253.0.0 mask 255.255.0.0 172.18.4.1 -p

:: 하이웨어
route add 192.203.138.101 mask 255.255.255.255 172.18.4.1 -p

:: 아래는 정확히 잘 모름 (정영은 부장이 보낸 테이블에 포함되어 있음)
route add 192.203.138.110 mask 255.255.255.255 172.18.4.1 -p
route add 192.203.138.11 mask 255.255.255.255 172.18.4.1 -p
```

- 아래에 게이트웨이 주소 입력 후, 명령어 CMD에 붙여넣기
- \*초기화 명령어 : route -f
- \*로컬 네트워크(127.0.0.1)이 안될 시, 다시 시작 실행
- \*[링크](https://blog.naver.com/devch/220805952113) : '3. 네트워크 인터페이스 번호 기억하기' 부분 참고
