<br>

- 요청받은 것
    - 일정관리 페이지 필요 (조기취업)
    - 출석부 페이지와 조기취업 연동(조기취업)
    - 교원서비스에서 메뉴를 교수가 볼 수 있도록 함 (성적경고자)

<br>

- 내용
    - 조기취업자 학점 인정 설명
        - 조기취업 학생들이 취업해서 학교를 못 올 때의 대체 교과목을 말함
        - 강의에 오지 않더라도 교과목 교수가 과제를 부여해서 출석을 인정함
        - 학생이 학점 인정 신청서 작성하여 출석 대체자료로 관리함
    - 필요성
        - 엑셀로만 하니까 관리가 안됨
        - 조기취업자가 성적 자료가 없어서 경고 받은 적이 있음
        - 후임자가 왔을 때 관리가 된 파일이 없어서 인수인계가 힘듦
    - 신청서류 - 출석인정신청서, 사대보험가입증명서, 재직증명서
        - 출석인정신청서 항목은 종이 서류의 항목 그대로 간다고 함
    - 금오공대 신청 프로세스
        - 학기 초 서류없이 신청만 받고, 학기 말에 서류를 제출함
    - 프로세스 요청사항
        - 수기로 작성된 학점인정신청서를 교수가 직접 받기를 원함
        - 신청기간을 관리할 수 있기를 원함 → 일정관리 페이지 필요
    - 신청 프로세스 논의
        - 승인 후 승인 취소가 필요한 경우가 있음 (학생이 F학점을 받는 경우, 학생이 힘들어서 취소한 경우, 학기 중 퇴사한 경우)
        - 학기 초 실제 재직을 안하고 있는데 신청만 해놓고, 학기 말 취업해서 서류를 올리는 가능성이 있기에 학기 초, 조기 승인과 학기 말, 최종 승인 시 각각 서류를 받자고 함. 이 경우 신청 취소하고자 하면 최종 승인을 하지 않으면 됨
        - 승인 책임

            - 1\. 교과목 교수, 신청자 학과, 교육혁신과가 승인 책임을 가짐

                - 이 경우, 승인을 가진 모든 곳에서 승인을 해야 처리되는지, 어느 한 곳만 승인되면 처리인지는 논의가 필요
            - 2\. 조기, 최종 승인은 주체가 교육혁신과이며 교과목 교수, 신청자 학과는 신청 현황을 볼 수 있음
                - 위 경우, 승인이 지연될 수 있고, 한 학기에 50명 정도여서 혼자서 처리 가능하기에 제안됨
                - 신청 및 첨부물 올라오면 알림 필요

        - 금오공대 프로세스로 갈지, 이번에 이야기한 프로세스로 갈지 논의한다고 함. → 추후 내용 요청
    - 신청 대상자
        - 4학년 1학기 학생 대상으로 함
    - 연동되는 화면이 있는지?
        - 출석부에 현재 공결 넘어가는 구조처럼 조기취업도 넘어가기를 원함
    - 기존에 있던 서류의 데이터를 전산에 넣고 싶다고 요청
        - → 입력 페이지 필요

<br>

- 성적경고자 상담
    - 신청서 양식 요청 → 추후 받아야 함
    - 통합 시스템 ‘교수상담’ - ‘학사경고자상담’ 탭 사용
    - 교수가 자의적으로 상담자를 선택함
    - 학습컨설팅 상담, 심리상담, 진로적성검사, 기타 가 있음
    - 진행상태 - 진행, 완료
        - 지꺼진 반란과 서로의 진행상태가 조회될 수 있도록 구성되야 함
    - 교원 서비스는 교수가 볼 수 있게끔
