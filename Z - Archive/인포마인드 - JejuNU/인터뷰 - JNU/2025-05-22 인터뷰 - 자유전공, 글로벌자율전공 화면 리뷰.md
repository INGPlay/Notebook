                - 휴학 → 3년 후에 단위가 달라질 수 있음
                    - 1학기 때는 있었다가 2학기에 사라지거나 이름이 바뀔 수 있고
                - 배정이 학기단위로 이루어져서 학기가 필요하다는 주장
                    - 학기단위로 일관성?
                - 학기단위로 학과이름이 바뀌거나 학과가 사라지지 않음
                - 자유전공이 역학기로 할 가능성
                - 입학기준이냐 복학기준이냐
                - 학과의 이름이 뭘로 바뀌는지 관리가 필요 → 코드도 바뀜
                    - 기존에는 각각 학생의 학과이름과 코드를 다 바꿔줌
                - 학기 단위로 학과가 바뀔 수도 있다 (지금까지는 거의 3월에만 바꿨지만)
                - 입학년도 기준 모집 → 현재로서는
                - 구학과를 조회할 수 있는 거 → 로그 테이블이 있으나
                - 학과 통합 있을 수 있음
                - 학과 변경 매핑 테이블 관리 어떻게 할 지 → 고민
                - 연도 → 입학연도, 신청연도 처럼 기준을 명확히 알 수 있게끔 변경 요청 받음
                - 포탈 → 대학이름은 풀네임으로 (대학이름 전공명)
                - 통합 → 희망전공을 희망대학, 희망학과, 희망전공 3개의 컬럼으로 분리
                - 희망전공신청 → 1학기(1차, 2월), 2학기(2차, 6월)
                    - 희망전공 2차를 해야 전공신청을 할 수 있다
                - 1차 희망전공신청은 입학전에는 수험번호로 받을 수 밖에 없다
                    - 임시로 수험번호로 조회할 수 있는 거를 만들어줘야 함
                - 자유전공 희망전공신청은 제한 없음 (1차, 2차, 3차, …)
                - 자유전공 희망전공신청에 1, 2, 3 지망 적도록하기
                - 다음에 전공신청 할 사람들에 대한 준비 → 희망전공신청
                - 학생들의 전공탐색, 학사관리 차원 → 희망전공신청
                    - 마지막에 희망전공신청 정보를 조회하는 게 필요한 듯
                - 한 학기에 한 번 희망전공신청 할 수 있다 (두 번 이상 하지 않음)
                - 차수는 필요없을 것 같고, 학기 별로 관리하는게…
                - 현재 자유전공으로 있는 휴학생들 보여주기 (재학, 휴학 상태 추가)
                - 연도기준 조회 상관없이 전체 추가
                - 전공신청
                    - 2학기 신청 11월, 학적이관 3월
                    - 1학기 신청 5월, 학적이관 8월
                - 승인 후 전공 변경 → 없다고는 못하지만
                - 2학년 1학기에 휴학하더라도 전공신청됨
                - 1학년도 전과 가능
                - 전과하면 전공신청은 하지 않는다
                - 학기를 볼 수 있을 만한 테이블… (학적 마스터 테이블의 학기 수를 볼 수 없다)
                    - 가학년, 가학기 정보…
                - 학기가 고정될 만한게 있는지 → 알아보기 (가학년?)
                - 학기를 다 다니고 휴학하면 전공신청되고, 학기를 다 다니지 못하고 휴학하면 전공신청이 안됨
